## 目标

- 高可用基线：优雅退出、readiness/liveness、结构化日志、可观测性扩展点
- 结构化工程：单一错误出口、依赖边界清晰、可测试、可演进

## 规则

### HTTP 与错误处理

- 只允许一个错误出口：使用 `app/http/middleware/error_handler.go` 的 `HandleError/ErrorHandler`
- 控制器与中间件返回 error 即可，不要自行拼装多套 JSON 错误格式

### 依赖注入与全局状态

- 业务层禁止直接使用 `config.GlobalConfig` 与 `database.DB`
- 通过 dig 注入 `*config.Config`、`*database.Connection`、`helpers.CacheService` 等依赖

### 日志

- HTTP access log 使用 zap 结构化字段，必须包含 `request_id`
- 业务日志统一走 `helpers.Logger`

### 迁移与生成代码

- SQLite schema 与迁移以 Atlas 为唯一来源：
  - schema：`database/schema.lt.hcl`
  - migrations：`database/migrations/atlas/sqlite`
- 生成代码目录 `internal/db/gen/**` 仅允许通过 bobgen 生成，不手改

### 外部依赖与测试

- 单元测试默认不依赖外部服务（Redis/MinIO/Postgres 等），需要外部依赖的测试用显式环境变量开启

## 本项目约束

- Atlas 使用本机已安装版本，不使用 Docker
- 当前重构阶段先跳过 Postgres
