# 中间件使用说明

本项目已集成 GoFiber 最常用的中间件，提供完整的生产环境功能。

## 已集成的中间件

### 1. 请求ID中间件 (RequestID)
- **功能**: 为每个HTTP请求生成唯一的请求ID
- **用途**: 日志追踪、调试、请求关联
- **配置**: 自动生成UUID格式的请求ID

### 2. 恢复中间件 (Recover)
- **功能**: 从panic中恢复，防止应用崩溃
- **用途**: 生产环境稳定性保障
- **配置**: 启用堆栈跟踪

### 3. 日志中间件 (Logger)
- **功能**: 记录所有HTTP请求的详细信息
- **格式**: `[时间] 状态码 - 方法 路径 延迟`
- **用途**: 请求监控、调试、审计

### 4. CORS中间件 (CORS)
- **功能**: 处理跨域资源共享
- **配置**:
  - 允许所有来源 (`*`)
  - 支持所有常用HTTP方法
  - 允许认证信息
  - 缓存时间24小时

### 5. 安全头中间件 (Helmet)
- **功能**: 添加安全相关的HTTP头
- **保护**: XSS、点击劫持、MIME类型嗅探等
- **用途**: 增强应用安全性

### 6. 压缩中间件 (Compress)
- **功能**: 自动压缩响应数据
- **算法**: 最佳速度压缩
- **用途**: 减少网络传输，提升响应速度

### 7. ETag中间件 (ETag)
- **功能**: 生成ETag用于HTTP缓存
- **用途**: 减少不必要的数据传输
- **优势**: 提升性能，节省带宽

### 8. 限流中间件 (Limiter)
- **功能**: 限制客户端请求频率
- **配置**:
  - 每分钟最多100个请求
  - 基于IP地址限流
  - 跳过健康检查和监控端点
- **保护**: 防止DDoS攻击和滥用

### 9. 监控中间件 (Monitor)
- **功能**: 提供实时系统监控
- **访问路径**: `/monitor`
- **信息**: 请求统计、响应时间、状态码分布
- **刷新**: 每秒自动刷新

## 特殊端点

### 健康检查
- **路径**: `/health`
- **功能**: 应用健康状态检查
- **响应**: `{"status": "ok"}`

### 系统监控
- **路径**: `/monitor`
- **功能**: 实时性能监控面板
- **特性**: API模式，每秒刷新

## 自定义中间件

### 错误处理中间件
- **404处理**: 自定义404错误响应
- **全局错误**: 统一的错误处理格式
- **日志记录**: 自动记录错误信息

### 认证中间件
- **JWT保护**: 现有的JWT认证中间件
- **扩展性**: 可添加更多认证相关功能

## 配置说明

所有中间件都可以通过修改 `app/middleware/setup.go` 文件进行自定义配置：

```go
// 示例：修改限流配置
app.Use(limiter.New(limiter.Config{
    Max:        200,                    // 增加到200个请求
    Expiration: 30 * time.Second,       // 缩短到30秒
    KeyGenerator: func(c *fiber.Ctx) string {
        return c.Get("X-API-Key")      // 使用API Key限流
    },
}))
```

## 性能优化建议

1. **生产环境**: 建议调整CORS配置为具体域名
2. **限流**: 根据业务需求调整限流参数
3. **日志**: 生产环境可考虑异步日志
4. **监控**: 定期清理监控数据

## 安全建议

1. **CORS**: 生产环境限制具体域名
2. **限流**: 根据用户级别设置不同限流策略
3. **安全头**: 定期更新Helmet配置
4. **监控**: 监控异常请求模式
